Cómo leer los resultados: por restricciones de seguridad del navegador (CORS), no siempre se puede leer el código de estado exacto (200, 404, etc.) de un sitio ajeno. "Responde" significa que el navegador logró conectarse al servidor y recibió algo — el sitio está en línea, aunque no garantiza que la página cargue perfecto. "Sin respuesta" casi siempre significa que el dominio no resuelve, el proyecto fue borrado, o el servidor está caído. Ante la duda, abrí el link manualmente con el ícono ↗.
